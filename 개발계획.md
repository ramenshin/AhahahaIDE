# devIdeTool 개발계획

## 1. 프로젝트 목적

`D:\Projects` 아래 수십 개의 프로젝트를 Claude Code + PowerShell 환경에서 작업할 때, 매번 터미널 창을 열고 디렉토리 이동, venv 활성화, claude 실행을 반복하는 번거로움을 제거한다.

IDE 스타일의 단일 GUI 애플리케이션으로 **좌측 프로젝트 탐색기 + 중앙 파일 탐색기 + 우측 탭형 터미널(수직 분할)** 3단 구조를 제공하여, 폴더 클릭만으로 해당 프로젝트의 파일 구조와 Claude Code 세션이 동시에 열리고 여러 세션을 자유롭게 오가며 작업할 수 있게 한다. 각 터미널 탭은 **Claude 실행 페인(상단 70%) + 일반 PowerShell 페인(하단 30%)**으로 수직 분할되어, Claude와 대화하면서 동시에 실행/테스트/Git 명령을 병행할 수 있다.

## 2. 기술 스택

**채택: Electron + TypeScript + React + xterm.js + node-pty**

| 컴포넌트 | 선택 | 이유 |
|---|---|---|
| 앱 프레임워크 | Electron | VS Code 통합 터미널과 동일 스택, Windows 검증 완료 |
| 언어 | TypeScript | 타입 안정성, node-pty/xterm 타입 지원 |
| UI | React | 파일 탐색기·탭·모달 UI 구현 용이 |
| 레이아웃 | react-resizable-panels 또는 Allotment | 리사이즈 가능한 3단 분할 |
| 터미널 렌더러 | xterm.js + addons (fit, search, web-links) | Claude Code TUI 지원 검증됨 |
| 터미널 백엔드 | node-pty | Windows ConPTY 네이티브 지원 |
| 파일시스템 감시 | chokidar | 프로젝트 내 파일 변경 감지 |
| Git 정보 | simple-git 또는 `.git` 직접 파싱 | 브랜치명, 변경 상태 |
| 빌드 | Vite + electron-builder | 개발 편의성, 포터블 exe 생성 |
| 상태 저장 | electron-store | 설정/세션 상태 persistence |
| 프로세스 정리 | tree-kill | 자식 프로세스 트리 일괄 종료 |
| 시스템 알림 | Electron Notification API | 백그라운드 세션 완료 알림 |

## 3. 화면 레이아웃 (3단 + 상태바)

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 상단 툴바: [+] 새 폴더  [⚙] 설정  [⌘P] Quick Switch  [⌘⇧P] 명령 팔레트    │
├────────────┬──────────────────┬──────────────────────────────────────────┤
│ ① 프로젝트 │ ② 파일 탐색기    │ ③ 터미널 영역                            │
│   트리     │  (선택 프로젝트)  │ ┌──────────────────────────────────┐ │
│            │                  │ │ [proj-a][proj-b•3][proj-c]       │ │
│ ★ 즐겨찾기 │  📁 src          │ ├──────────────────────────────────┤ │
│  ● proj-a  │   📄 index.ts    │ │ ▲ Claude (70%)                   │ │
│  ● proj-b  │   📄 app.tsx     │ │   (venv) PS> claude              │ │
│            │  📁 tests        │ │   > 코드 리뷰해줘                 │ │
│ 최근       │  📄 README.md    │ │   ● Read(src/main.py)            │ │
│  ○ proj-c  │  📄 package.json │ │   ...                            │ │
│            │                  │ ├──────── resize bar ──────────────┤ │
│ 전체       │                  │ │ ▼ PowerShell (30%)               │ │
│ 🟢 proj-a  │                  │ │   (venv) PS> python main.py      │ │
│ 🟡 proj-b  │                  │ │   [INFO] OK                      │ │
│ ⚪ proj-d  │                  │ │   (venv) PS> _                   │ │
│ 🔴 proj-e  │                  │ │                                  │ │
│            │                  │ │                                  │ │
│ ✎ 세션메모 │                  │ └──────────────────────────────────┘ │
├────────────┴──────────────────┴──────────────────────────────────────────┤
│ 상태바: 세션 12/20 · RAM 4.2GB · 활성 3 · 📁 D:\Projects\proj-a · (main)  │
└──────────────────────────────────────────────────────────────────────────┘
```

**패널 동작**
- 3개 패널 모두 **좌우 리사이즈 가능** (드래그 바)
- 각 패널 **접기/펼치기 토글** 가능 (단축키: `Ctrl+B` 좌측, `Ctrl+E` 중앙)
- 기본 비율: 18% · 22% · 60% (16:9 기준)
- 최소 너비 제약: 좌측 180px, 중앙 200px, 우측 400px
- 레이아웃 상태는 electron-store에 자동 저장/복원

**패널별 역할**
- **① 프로젝트 트리**: `D:\Projects` 하위 프로젝트 폴더 목록. 즐겨찾기/최근/전체 섹션. 상태 아이콘, Git 브랜치, 열린 상태 색상 강조. 우클릭 메뉴.
- **② 파일 탐색기**: 현재 활성 탭 프로젝트의 파일/폴더 트리. chokidar로 실시간 반영. 파일 우클릭 시 액션 메뉴.
- **③ 터미널 영역**: 상단 탭 바 + 수직 2분할 터미널.
  - **상단 페인 (70%)**: PowerShell + venv + claude 실행
  - **하단 페인 (30%)**: PowerShell + venv (일반 실행/테스트/Git용)
  - 두 페인 모두 동일 폴더에서 독립 PTY로 실행 (venv 활성화됨)
  - 페인 간 분할 비율 드래그로 조정 가능, 설정에서 기본값 변경 가능
  - 포커스된 페인은 좌측 accent 색 표시
  - `Ctrl+Tab` / `Ctrl+\`` 등 단축키로 페인 간 포커스 이동

## 4. 아키텍처 개요

```
┌─────────────────────────────────────────────────────────┐
│                     Electron Main                       │
│  - Window / Tray 관리, IPC 라우팅                        │
│  - node-pty 프로세스 풀 (프로젝트별 세션)                 │
│  - 파일시스템 작업 (폴더 추가/삭제/이름변경)              │
│  - chokidar: 활성 프로젝트 파일 트리 감시                │
│  - Git 브랜치 조회, 자동 저장, Hibernate 타이머           │
│  - 시스템 알림 발송                                      │
└─────────────────────────────────────────────────────────┘
                 ▲
                 │ IPC
                 ▼
┌─────────────────────────────────────────────────────────┐
│                     Renderer (React)                    │
│  ┌──────────┬──────────────┬────────────────────────┐  │
│  │ 프로젝트  │ 파일 탐색기   │ 터미널 탭 + xterm      │  │
│  │ 트리     │              │                        │  │
│  └──────────┴──────────────┴────────────────────────┘  │
│  ┌──────────────────────────────────────────────────┐  │
│  │              상태바 (세션/RAM/경로)                │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**세션 생명주기**
1. 좌측 폴더 클릭 → 세션 없으면 신규 생성(최대 세션 체크), 있으면 활성 탭 전환
2. 중앙 파일 탐색기에 해당 프로젝트 파일 트리 렌더, chokidar 감시 시작
3. 우측에 PowerShell 스폰 → `cd <folder>` → venv activate → `claude` 실행
4. 유휴 N분 초과 → claude 자동 hibernate (PowerShell 유지)
5. 탭 닫기 → tree-kill로 프로세스 트리 종료, chokidar 감시 해제
6. 앱 종료 → 활성 세션 있으면 확인 → 상태 저장 → 정리 후 종료

## 5. 기능 요구사항

### 5.1 핵심 기능 (확정)
- 3단 레이아웃 (프로젝트 트리 / 파일 탐색기 / 터미널)
- **단일 루트** 방식 (rootPath는 설정에서 변경 가능, 기본 `D:\Projects`)
- 폴더 선택 시 파일 트리 + 터미널 수직 2분할 (claude 70% + 일반 PS 30%) 자동 실행
- 탭형 다중 세션 관리
- 열린 폴더 시각 표시 (색상 + 상태 아이콘)
- 설정 화면: 최대 세션 수, 루트 경로, Hibernate 시간, 시작 모드 등
- 최대 세션 초과 시 에러 팝업 모달
- 폴더 우클릭 메뉴: 열기 / 닫기 / 경로 복사 / 탐색기·VSCode로 열기 / 새 폴더 만들기 / 이름 변경 / 삭제
- 주기적 상태 저장 (5초 debounce)
- 시작 모드: (A) 빈 상태 / (B) 마지막 상태 복원 (세션 자동 재시작)

### 5.2 파일 탐색기 (중앙 패널)
- 활성 탭 프로젝트의 파일/폴더 트리 표시
- chokidar로 실시간 반영 (파일 추가/삭제/이름변경 즉시 갱신)
- 제외 패턴 설정 가능 (`node_modules`, `.git`, `venv`, `__pycache__` 등)
- 파일 우클릭 메뉴:
  - 경로 복사 (절대)
  - 프로젝트 기준 상대 경로 복사
  - 터미널에 경로 삽입 (`@경로` 형태로 Claude에 전달 가능)
  - 기본 에디터로 열기
  - VS Code로 열기
  - 탐색기에서 보기
- 폴더 우클릭 메뉴: 새 파일 / 새 폴더 / 이름 변경 / 삭제
- 검색창(`Ctrl+Shift+F`)으로 파일명 필터
- 탭 전환 시 파일 트리 자동 교체

### 5.3 백그라운드 세션 인지
- **탭 배지**: 비활성 탭에 새 출력 발생 시 숫자 배지 (99+ 상한)
- **완료 감지**: claude 응답 완료 후 입력 대기 진입 시 탭 깜빡임
- **시스템 알림**: 백그라운드 완료 시 Windows 알림 (클릭 시 해당 탭 포커스)
- 알림 on/off 설정

### 5.4 빠른 탐색
- **Quick Switch (`Ctrl+P`)**: 프로젝트명 퍼지 검색 → 즉시 탭 전환/열기
- **즐겨찾기 핀**: 상단 고정, 드래그 정렬
- **최근 사용**: 최근 10개 자동 섹션

### 5.5 리소스 관리
- **하단 상태바**: `세션 N/MAX · RAM X.XGB · 활성 N · 경로 · 브랜치`
- **자동 Hibernate**: 유휴 시 claude만 종료 (PowerShell 유지)
  - 설정: off / 10분 / 30분(기본) / 1시간 / 2시간
- **수동 Hibernate**: 탭 우클릭 → "Hibernate"

### 5.6 폴더 트리 정보 강화
- **상태 아이콘**: 🟢 활성 / 🟡 Hibernate / 🔴 오류 / ⚪ 미실행
- **Git 브랜치**: 폴더명 우측에 회색 `(main)` 표시, 주기적 refresh
- **마지막 활성 시간**: 툴팁에 "3분 전 활성"
- **열린 폴더 색상 강조**: 밝은 primary 컬러, 나머지는 회색

### 5.7 안전성
- **종료 확인**: 활성 세션 있을 때 "N개 세션 실행 중. 종료?"
- **세션 로그 자동 저장**: `logs/<project>/<yyyy-mm-dd>.log` (최근 7일 롤링)
- **크래시 감지 & 재시작**: claude 비정상 종료 시 탭 내 "재시작" 버튼 + 오류 아이콘
- **파일 삭제**: 2단계 확인, 기본 휴지통(`shell.trashItem`), 영구 삭제는 추가 확인

### 5.8 편의 기능 (선택기능 모두 포함)
- **명령 팔레트 (`Ctrl+Shift+P`)**: "모든 세션 종료", "모두 hibernate", "설정 열기", "로그 폴더 열기", "레이아웃 초기화" 등
- **프롬프트 스니펫**: Claude 프롬프트 템플릿 저장 → 단축키로 터미널 삽입, CRUD UI
- **세션별 메모**: 각 탭의 "작업 메모" (좌측 하단 또는 사이드 패널)
- **폰트/테마 커스터마이징**:
  - 터미널 폰트 패밀리/크기
  - xterm 색상 테마 프리셋 (Dark/Light/Solarized/Dracula)
  - **UI 테마 프리셋** (설정 → 테마, 기본값 = 틸 그린):
    - A. 스왑 (퍼플/블루)
    - B. VS Code 활동바
    - C. 미드나잇 블루
    - **D. 틸 그린 (기본)**
    - E. Dracula
    - F. One Dark
    - G. Solarized Dark
    - H. Gruvbox Dark
    - (사용자 커스텀 색상 저장 v1.1)
- **탭 드래그 정렬** / **탭 그룹 색상** (분류 라벨)
- **전체화면 (`F11`)**, **터미널 줌 (`Ctrl +/-`, `Ctrl+0`)**

### 5.9 폴더 관리
- **새 폴더 생성**: 상단 `+` 버튼 또는 우클릭 → 다이얼로그 (이름, 위치, 선택 템플릿)
- **이름 변경**: 우클릭 또는 `F2`. 활성 세션 있으면 자동 종료 → 이름 변경 → 필요 시 재시작
- **폴더 삭제**: 우클릭 → 2단계 확인. 활성 세션 자동 종료 후 삭제

## 7. 핵심 기술 고려사항

1. **ConPTY**: Windows 10 1809+ 기본 지원
2. **node-pty 네이티브 바인딩**: Electron 버전 매칭 prebuilt (`@electron/rebuild`)
3. **Claude Code TUI**: xterm.js에서 정상 렌더링 (VS Code에서 검증됨)
4. **프로세스 트리 정리**: `tree-kill` 또는 `taskkill /PID /T /F`
5. **Hibernate 구현**: claude 자식 PID만 선택 kill, PowerShell 유지 → 재진입 시 `claude` 재입력
6. **백그라운드 출력 감지**: xterm.js `onData`로 비활성 탭 카운트
7. **완료 상태 감지 (heuristic)**: 프롬프트 재표시 + 일정 시간 신규 출력 없음
8. **Git 브랜치**: `simple-git` 또는 `.git/HEAD` 파싱, 주기적 refresh
9. **chokidar 성능**: 대형 `node_modules` 등 제외 패턴 필수, 활성 프로젝트 1개만 감시
10. **파일시스템 작업**: 삭제는 `shell.trashItem` 우선, 영구는 `fs.rm`
11. **자동 저장 debounce**: 5초, 변경 없으면 skip
12. **레이아웃 persistence**: 패널 비율/접힘 상태 electron-store 저장
